학습 전략에 대해서..

이 문서는 학습 전략, 자기 자신은 어떻게 학습을 하는지 등에 대해서 설명하고 있습니다.


1. 개요

학습 전략이란 학습을 하기 위해서 학습하는 동안 학습자가 취하는 모든 방법적 사고와 행동을 의미한다.

또한, 이러한 행동은 학습 과제나 새로운 정보를 보다 효과적으로 처리하는데 필요한 자기 통제기술이며, 자기 관리 기술, 수업 참여 기술, 쓰기 기술 등으로 구성되어 있다.

학습 전략의 대표적인 요소들
자기관리 기술 학습자료 및 공간의 조직, 시간 관리, 스트레스 관리, 학습에 필요한 도움 구하기, 컨디션 잘 조절하기(?) 등
수업참여 기술 수업참여 및 청강하기, 강의 내용 노트하기 등
읽기 기술 개관, 요점 정리와 이해, 이해 점검과 질문 등
쓰기 기술 주제 정하고 분류하기, 개요 작성하기 등
시험 관리 기술 질문내용 검토, 시험시간 사용, 시험 불안에 대한 심리적 대처 등
기타 기술들 시간 관리, 학습 습관 등

2. 자기 자신은 어떻게 학습하는가?

자신이 중요하다고 생각하는 학습 전략 TOP 5
정리하기 예습, 복습하기 실습자료 활용해보기 컨디션 조절 시간 관리

2.1. 수업할 때

수업할 때는 실습 같은 것이 있으면 해보고 하다가 모르는 것이 있으면 질문을 하고 설명을 듣는 경우, 중요한 부분이 어디인지 잘 듣고, 책에 있는 해당 부분에 형광펜 혹은 볼펜 등으로 표시를 해둔다. 또한 책에 내용이 없어서 필기를 해야하는 경우에는 책이나 노트 등에 필기를 하거나 필기가 조금 힘들다 싶으면 미리 사진을 찍어놓고 집에 와서 필기를 한다.

근데 정작 본인은 집에 와서 시험 기간 때 말고는 복습을 안 한다는 게 함정... 그렇지만 숙제는 안 까먹고 잘 해오는 편이다.


2.2. 집에서 공부할 때

집에서는 학교에서 수업 중에 밑줄 등으로 표시한 부분, 책이나 노트 등에 필기한 내용들을 보면서 외우고 정리할 때가 많다. 그렇지만 시험이 수업 진행 중에 풀었던 문제에서 주로 나오는 경우도 있어서 외우고 정리하고 난 후에는 문제를 한 번 다시 풀어본다. 그렇지만 시험 문제가 책 문제와 꼭 똑같이 나오지는 않아서 문제를 푼 다음에도 몇 번 다시 정리해보는 등 복습도 한다.

수업했던 자료들이 이 홈페이지에 있는 경우에는 홈페이지에 들어가서 해당 ppt 혹은 pdf 자료 등을 집에 있는 컴퓨터나 휴대폰 등으로 미리 받아놓는다.

또한 일부 과목은 실습이 많은 과목도 있는데, 이러한 과목은 책이나 위의 수업 자료에 있는 실습 내용을 보고 다시 한번 실습해보고 응용해보기도 한다. (그렇지만 결과가 이상한 경우에는 어디가 잘못되었는지 꼭 확인해야 한다.)

그리고 다음 날 아침에 일어나기 힘들다고 한다.

3.1. 객체지향 언어

객체지향 언어
수업 시간 월요일 1~4교시
난이도 ★★★☆☆
실습 존재 여부 O
사용하는 프로그램 Visual Studio (설치하기)

이 과목은 C++ 언어에 대한 이론보다는 이를 이용해서 프로그램을 만든 다음에 출력을 시키는 내용(실습)이 많이 존재한다. 물론 어떻게 프로그램을 만들어 내는지에 대해서 한 번 생각해보고 여기 나오는 프로그램 설계 방법을 단순히 책을 보고 외우기도 하지만, 주로 이 과목을 학습할 때는 프로그램 설계를 컴퓨터로 직접 해보는 경우가 많다.

또한, 책에 나온 프로그램들을 약간 변형시켜서 한 번 설계를 해보기도 하는 편이다.

여기서는 가장 힘든 부분인 함수가 나오는 부분을 위주로 학습을 한다.

여기에서 사용되는 구성 요소들은 예전 C언어나 현재 자료구조 시간 때 사용되는 프로그램 구성 요소와는 약간 다르기 때문에 실수하지 않도록 주의해야한다.




▲ 자료구조 수업 내용(왼쪽 사진)과 객체지향 언어 수업 내용(오른쪽 사진), 객체지향 언어 과목에서는 printf, scanf와 같은 구성 요소 대신에 cout, cin과 같은 구성 요소를 사용한다.



3.2. 데이터베이스

데이터베이스
수업 시간 화요일 2~4교시
난이도 ★★☆☆☆
실습 존재 여부 X
사용하는 프로그램 없음

이 과목은 딱히 컴퓨터를 이용한 실습은 없고, 전부 이론이다. 따라서 책에 나와있는 내용들 중에서 중요하다고 생각되는 부분을 외우고 정리한 다음에 문제를 몇 번 정도 다시 풀어보는 방식으로 학습을 진행하고, ppt와 같은 수업 자료들도 필요하면 다운 받아서 참고한다.

그렇지만 시험에 대부분 책에 나온 문제들과 비슷한 문제들이 나오는 편이라 학습 할 때 문제 푸는 것 위주로 하는 편이다.

그래도 복잡한 부분은 많이 있지는 않은 편이라 학습 자체는 어렵지 않다.


3.3. 자료구조

자료구조
수업 시간 월, 화요일 6~7교시
난이도 ★★★★☆
실습 존재 여부 O
사용하는 프로그램 Visual Studio (설치하기)

이 과목도 객체지향 언어와 비슷하게 프로그램을 어떻게 설계하는지에 대해서 외우거나 정리하기보다는, 컴퓨터를 이용하여 프로그램을 작성, 테스트 해보고, 이러한 프로그램도 약간 변형시켜서 몇번 정도 테스트 해보기도 한다.

이 과목의 경우, 트리나 이중 연결 리스트 혹은 해싱과 같이 이해하기에 시간이 오래 걸리는 부분이 어느 정도 있다. 따라서 이 과목이 난이도가 있는 만큼, 시험 기간 때 특히 신경을 써야 한다고 본다. 말로만?

주로 이 과목을 학습할 때는 아래와 같은 수업할 때 사용되는 pdf 파일을 참고한다.



자료구조 수업 자료(8장 트리) 보기 (단, 익스플로러에서는 실행이 잘 안될 수 있음.)




3.4. 토익 I

토익 I
수업 시간 수요일 2~3교시
난이도 ★★☆☆☆
실습 존재 여부 X
사용하는 프로그램 없음

이 과목의 경우, 어떤 형태의 단어를 써야 하는지, 그리고 이러한 모습의 단어들(ex. to나 ing가 붙어있는 동사 등등)은 어떤 특징을 가지고 있는지, 그리고 분사 구별 등 구별 방법은 어떤 것인지 외우고 정리한 다음, 문제를 풀어보고, 몇 번 복습을 한다.

그 중에서 문맥에 맞는 것을 찾는 경우에는 단어를 외우기도 하며 문제를 풀 때 지문이 있는 경우에는 지문을 읽어보고 문제를 푼다. (단 시간이 부족한 경우에는 문맥상 맞는 문장 또는 단어 찾는 부분 한정이며, 다른 경우에는 빈 칸이 있는 문장을 확인해보는 편이다.)

이 과목의 학교 시험은 책에서 봤던 문제들 위주로 나오는 편이라 책에 있는 문제들을 잘 이해하는 것이 중요하다.


3.5. 서버 구축 및 관리

서버 구축 및 관리
수업 시간 수요일 6~9교시
난이도 ★★★☆☆
실습 존재 여부 O
사용하는 프로그램 VMware workstation-pro (설치하기)
추가로 설치할 것 Ubuntu-16.04-desktop-amd64.iso (설치하기)

이 과목의 경우, 컴퓨터 가상 머신의 터미널에서 입력하는 명령어에는 어떤 것이 있고, 그 명령어가 어떤 역할을 하고, 여기서 사용되는 파일(ex. /etc/hosts 등)은 어떤 파일인지, 그리고 서버 운영은 어떻게 하는지 등에 대해서 이해해보고, 책에 나와있는 실습 내용들을 보고 테스트를 해본다.

또한, 여기서도 구현 순서나 구성도나 흐름도에 대해서는 몇 번 외우고 정리해본다.

이 과목은 명령어를 입력하는 것과 명령어의 특징을 위주로 학습을 진행하는 편이다.


3.6. 인터넷 활용

인터넷 활용
수업 시간 목요일 1~4교시
난이도 ★★☆☆☆
실습 존재 여부 O
사용하는 프로그램 Sublime text 3 (설치하기)

이 과목의 경우에는 HTML 또는 CSS 등의 사용법을 이해한 다음에 이 내용들을 한 번 테스트 해본 다음에, 여러가지 HTML 파일을 만들어보는 등 활용을 해본다.

이 과목의 HTML, CSS 부분은 테스트 또는 활용을 어느 정도 해봐서 많이 어렵지는 않지만 Javascript의 내용이 있는 경우에는 조금 힘들기도 하다.

따라서 여기서는 Javascript에서 사용되는 것과 같이 이해하기 힘든 것 위주로 학습을 진행하며, 가끔씩 네이버와 같은 인터넷 또는 HTML 또는 CSS 등을 다루는 책을 한 번 보기도 한다.

여기에 나오는 HTML, CSS, Javascript파일을 만들 때 어떻게 하는지 알 수 있는 사이트는 대표적으로 w3schools.com 이라는 사이트가 있다.


4. 결론

수업을 듣는 과목 중에서는 실습하는 내용이 있는 과목도 있고 없는 과목도 있다. 한 번 실습을 해봐야 하는 과목들은 실습 내용을 위주로, 실습 내용이 없는 과목은 책에 나와있는 내용을 외우고 정리해보고 문제를 풀어보는 것 위주로 학습을 진행한다.

따라서 책에 나온 내용, 또는 문제 풀이 위주로 하는 과목과 실습 내용을 위주로 하는 과목을 분류해보면..

5. 기타 내용

물론 학습을 진행하는 것도 좋지만, 너무 무리한 학습 진행은 오히려 효과가 없을 수도 있다.

너무 늦은 시간까지 무리한 학습을 진행하는 것은 오히려 집중력이 떨어뜨리고 건강에도 피해를 끼칠 수 있다. 그러므로 피곤하다 싶으면 그냥 자고 다음날 기분 좋게 하루를 시작하는 것이 훨씬 낫다.

위에 기분 좋게는 아닌데..

또한, 새벽이나 이른 아침에 무리하게 학습 진행을 시작하는 것도 좋지는 않다.


만약 시험 기간 때처럼 굳이 밤을 새야하는 경우에는 커피나 단 음식 등을 먹으면서 학습을 진행해도 괜찮다. 그렇지만 컨디션 조절은 더욱 신경을 써야한다. 전날 열심히 공부했는데 시험 때 피곤하면 그 날 시험은 거의 망하는거다.


마지막으로, 학습 진행 중에는 운동 및 여가 활동 등을 휴식 시간을 갖는 것도 중요하다. 휴식 시간을 가지면서 스트레스를 풀어가는 것도 학습 전략 중에 하나라고 본다. 학업으로 인해 스트레스를 많이 받으면 정신적, 신체적으로도 해롭기 때문에, 그렇지만 시험 바로 전에 너무 놀아서는 안되는 건 당연하고..


그래도 학습 전략 중에서 가장 중요한 것은 시간 관리라고 본다.
